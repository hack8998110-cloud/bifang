# 毕方 V0.2-dev 版本说明

状态：开发中，不作为正式发布版。

## 版本目标

V0.2-dev 的目标不是增加更多拍剪建议，而是强化毕方的编导判断能力：

- 让客户先知道“用户想看什么”。
- 让客户先知道“今天该拍什么”。
- 让客户知道“为什么先拍这条”。
- 让客户改脚本时不改丢用户视角、判断标准、信任证据和成交理由。

## 与 V0.1 的区别

V0.1 重点是建立可用链路：

```text
诊断 -> 建档 -> 选题 -> 脚本 -> 审核 -> 改稿 -> 复盘
```

V0.2-dev 重点是建立版本门槛：

```text
行业输入 -> 用户判断题 -> 先拍优先级 -> 改稿锚点 -> 跨行业压测
```

## 开源线

开源版 V0.2-dev 只加入通用方法：

- 快速开始。
- 全行业示例。
- 编导思维压测矩阵。
- 开源边界说明。
- 通用失败回归用例。

不加入：

- 私有达人蒸馏。
- 竞品账号研究。
- 客户真实数据。
- 内测销售和交付资料。

## 内测线

内测版 V0.2-dev 可以继续加入私有增强：

- 达人内容蒸馏后的内部判断规则。
- 真实客户样本压测。
- 私有行业资产库。
- 账号复盘记录。
- 服务方交付流程。

这些内容不进入开源版。

## 发布门槛

满足以下条件后，才考虑打 `V0.2-beta`：

- 至少 20 个行业样本完成压测。
- 低信息输入下，默认不直接写脚本。
- 每个样本都能输出用户判断题、先拍理由和改稿锚点。
- 高风险行业不出现疗效、收益、保证、绝对化承诺。
- 开源包无私有研究、内测文档、销售资料和工具污染。
