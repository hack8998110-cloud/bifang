# Bifang Smoke Cases

这些不是严格单元测试，而是人工/模型自检样例。修改 skill 后，用这些输入检查默认输出是否跑偏。

## Case 1: 本地餐饮新号

输入：

```text
我开一家社区面馆，主打现熬牛肉汤面，客户是附近上班族和小区居民，想通过抖音带来到店。
现在有门头、出餐过程、后厨熬汤画面，没有顾客采访。
```

期望：

- 先说用户真正想看的不是“面有多好吃”，而是“这家店值不值得今天去试”。
- 判断用户决策阶段：附近用户准备行动，但缺少试店理由。
- 给 3 条先拍内容。
- 第一条有“用户问题/判断标准/信任证据/成交理由/承接动作”。
- 不默认输出完整 60 秒脚本。

## Case 2: 高客单服务

输入：

```text
我做全屋定制，客户是准备装修的业主，想通过小红书获得私信咨询。
我有过往案例照片和板材样品，但不能公开客户姓名。
```

期望：

- 先说用户想看的是“怎么判断不踩坑”，不是商家套餐介绍。
- 判断用户决策阶段：装修前正在比较，缺判断标准和可信证据。
- 强调预算、材料、工艺、案例打码。
- 不承诺效果，不虚构客户结果。

## Case 3: 老账号复盘

输入：

```text
我的账号发了三个月，有几条播放不错，但私信很少，帮我复盘一下。
```

期望：

- 不直接当新号出 3 条选题。
- 先要 9 条历史内容。
- 引导转 `bifang-feedback` 或 `bifang-report`。

## Case 4: 高风险行业

输入：

```text
我做口腔诊所，想拍种植牙内容，客户是缺牙人群，想通过视频获得预约咨询。
```

期望：

- 不制造恐吓，不承诺疗效、价格、成功率。
- 优先输出判断标准、流程透明、材料清单、面诊边界。
- 提醒案例和隐私需要打码。

## Case 5: 批量选题

输入：

```text
帮我给一家宠物店出 20 条选题，目标是到店洗护和复购。
```

期望：

- `starter` 不自己批量展开。
- 应转 `bifang-topic`。
- 如果先给简短建议，只能说明内容方向和转入模块。

## Case 6: 编导帮客户做内容

输入：

```text
我是短视频编导，帮一个建材老板做账号。他卖的是系统门窗，目标用户是准备装修的业主，想通过视频获得私信咨询。
客户有展厅、安装现场、材料样角，没有成熟口播能力。
```

期望：

- 先识别使用者是“帮客户做内容”。
- 不把目标用户写成同行编导或代运营。
- 先判断装修业主真正想看的是“怎么判断门窗别买错”，不是老板介绍品牌。
- 给出“为什么这样判断”，方便编导说服客户不要改回卖点介绍。
- 给 3 条先拍方向，并说明每条为什么先拍。

## Case 7: 有专业经验但不直接卖货

输入：

```text
我是一个职场培训老师，想做个人账号，主要讲新人沟通和汇报能力，目标是建立信任并引导课程咨询。
```

期望：

- 能处理“专业经验/课程/IP”而不只按门店或产品号处理。
- 先判断目标用户想看的是具体场景和可执行方法。
- 避免承诺升职加薪或保证结果。

## Case 8: 选题进入脚本

输入给 `bifang-script`：

```text
选题：装修业主第一次看系统门窗，别先问品牌，先看这 3 个位置
目标用户：准备装修的业主
痛点：怕花高价买到不隔音、漏风、安装差的门窗
用户为什么会看：这是装修前容易踩坑但普通业主看不懂的问题
用户看完拿走什么：知道看型材截面、密封结构、安装收口这 3 个判断点
成交理由：懂判断标准后更愿意私信发户型和预算咨询
可信证据：展厅样角、安装现场、材料截面；客户案例需打码
平台：小红书
```

期望：

- 脚本输出包含 `用户为什么会看` 和 `用户看完拿走什么`。
- 脚本输出包含 `改稿锚点`。
- 不把“系统门窗很好”写成主线，而是围绕业主判断标准写。

## Case 9: 审核打回

输入给 `bifang-review`：

```text
今天给大家介绍一下我们家的系统门窗。我们团队非常专业，服务过很多客户，质量好，价格也实惠，欢迎咨询。
```

期望：

- 用户观看闸门不过。
- 打回原因包含：用户为什么看不清、没有判断标准、信任来源空泛、广告感强。
- 不直接帮用户改文案。

## Case 10: 改稿不能改丢

输入给 `bifang-rewrite`：

```text
原文案：今天给大家讲一下系统门窗怎么选。第一要看品牌，第二要看价格，第三要看服务。我们很专业，欢迎咨询。
改稿锚点：
- 用户问题：装修业主怕门窗买贵还不隔音
- 判断标准：看型材截面、密封结构、安装收口
- 信任证据：展厅样角、安装现场、材料截面
- 成交理由：用户能发户型和预算来做初步判断
- 承接动作：私信发户型图和预算范围
```

期望：

- 改稿后保留 5 个锚点。
- 开头不再是“今天给大家”。
- 不改成另一个无关选题。
