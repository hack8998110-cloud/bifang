# 毕方开源发布验收清单

> 用途：发布前人工验收。目标不是证明毕方会写脚本，而是证明它能让陌生用户快速理解“用户想看什么、先拍什么、为什么先拍”。

## A. 第一次使用

输入：

```text
我开社区面馆，想拍抖音引流到店。
```

必须通过：

- 默认使用 `bifang-starter` 或等价三段式。
- 显式出现 `行业翻译成的用户判断题`。
- 输出只有“用户真正想看的 / 今天先拍 3 条 / 第一条别改丢”为主体。
- 不默认输出完整脚本、拍剪清单、评分表。

## B. 服务方使用

输入：

```text
我是编导，客户做全屋定制，只想要脚本。
```

必须通过：

- 识别使用者是编导，目标用户是装修业主。
- 输出能帮助编导向客户解释为什么先拍用户判断题，不是先拍老板卖点。
- 第一条别改丢包含用户问题、判断标准、信任证据、成交理由、承接动作。

## C. 高风险行业

输入：

```text
我做口腔诊所，想拍种植牙。
```

必须通过：

- 不承诺疗效、价格、成功率。
- 用户判断题围绕适不适合、流程、风险、材料和评估边界。
- 不直接进入成交脚本。

## D. B2B/设备

输入：

```text
我做工业除尘设备，想获客。
```

必须通过：

- 先围绕工况、适配、交付边界和售后判断。
- 不虚构参数、型号、效果数字。
- 承接动作是补工况信息，不是直接报价成交。

## E. 老账号复盘

输入：

```text
我发了半年，有播放但没有私信。
```

必须通过：

- 不直接当新账号出 3 条。
- 先要 9 条历史内容：播放最高 3 条、私信/成交最多 3 条、无效但常发 3 条。
- 判断问题指向观看理由、信任障碍或承接障碍。

## F. 失败回归

至少抽查 `failure_regression_cases.md` 中 FR-01、FR-03、FR-05、FR-07：

- FR-01 行业风险错配必须打回。
- FR-03 `qualityProblems` 非空但通过必须打回。
- FR-05 `XX/待填/占位` 证据必须打回。
- FR-07 诊断式 CTA 必须打回。

## 发布门槛

- `npm.cmd run v4:check` 通过。
- 以上 A-F 全部通过。
- README 中 Quick Start 与 `bifang-starter` 默认输出一致。
- `.codex/skills` 安装版与 `work/bifang_merged_latest` 源版本同步。
