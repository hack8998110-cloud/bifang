# 毕方 V0.2 压测矩阵

用途：判断毕方是否能在低信息、跨行业、真实客户场景里稳定给出编导思维，而不是直接套标题、写脚本或给拍剪建议。

## 总验收

每个样本默认只看 6 件事：

| 维度 | 合格标准 |
|---|---|
| 用户判断题 | 能把行业/产品翻译成用户正在判断的问题 |
| 决策阶段 | 能判断用户处于没意识到、正在比较、不信任或准备行动 |
| 先拍理由 | 每条先拍方向都说明为什么现在值得拍 |
| 三条分工 | 3 条不是同一种标题，至少覆盖判断、信任、行动中的两类 |
| 改稿锚点 | 第一条有用户问题、判断标准、信任证据、成交理由、承接动作 |
| 风险边界 | 不虚构证据，不跨行业套风险词，高风险行业不承诺结果 |

低于 5 项合格，视为样本失败。  
出现虚构数据、默认完整脚本、默认拍剪清单、高风险承诺，直接失败。

## 20 个行业样本

### 1. 社区餐饮

```text
我开社区面馆，想拍抖音引流到店。
```

期望：用户想判断“今天值不值得去试”，第一条优先拍试店理由和真实过程。

### 2. 烘焙门店

```text
我开一家社区烘焙店，想让附近宝妈和上班族来买早餐和生日蛋糕。
```

期望：用户想判断“是否新鲜、是否适合今天买、是否适合送人/孩子吃”，不能写健康功效承诺。

### 3. 宠物店

```text
我开宠物店，主要做洗护和寄养，想做同城号。
```

期望：用户想判断“宠物会不会被粗暴对待、环境是否卫生、流程是否放心”。

### 4. 系统门窗

```text
我卖系统门窗，想通过小红书拿装修客户咨询。
```

期望：用户想判断“怎么避免买贵、漏风、不隔音、安装翻车”。

### 5. 全屋定制

```text
我是全屋定制老板，有展厅和案例图，但客户不让露脸。
```

期望：用户想判断“板材、设计、安装和售后怎么避坑”，案例需打码。

### 6. 装修设计

```text
我做旧房改造设计，想通过视频拿业主咨询。
```

期望：用户想判断“自家房子适不适合改、预算和风险在哪里”，不默认承诺改造效果。

### 7. B2B 工业设备

```text
我们做工业除尘设备，想通过短视频获客。
```

期望：用户想判断“设备是否适配我的工况、供应商是否靠谱、售后边界是什么”。

### 8. 企业软件

```text
我们做进销存系统，想拍视频找中小商家客户。
```

期望：用户想判断“现在混乱是不是系统能解决、迁移成本和使用门槛多高”。

### 9. 企业增长咨询

```text
我做企业增长咨询，想拍视频拿线索。
```

期望：用户想判断“咨询师能不能先判断我的问题，而不是只讲理论”。

### 10. 职场培训

```text
我是职场培训老师，想卖沟通汇报课。
```

期望：落到具体职场场景，不承诺升职加薪。

### 11. 老板 IP

```text
我是一个做供应链十年的老板，想做个人 IP，未来接咨询和资源合作。
```

期望：用户想看行业判断、真实经验和可验证细节，不是老板自我介绍。

### 12. 手工定制

```text
我是做手工银饰定制的，想卖礼物款。
```

期望：用户想判断“值不值、适不适合送人、定制过程是否靠谱”。

### 13. 农产品

```text
我家卖平谷大桃，想拍视频卖货。
```

期望：用户想判断“甜不甜、熟度、发货、坏果处理”，不能写医疗/护肤风险词。

### 14. 日用洗护

```text
我做手工皂，想拍小红书卖产品。
```

期望：围绕使用场景、成分顾虑和清洁边界，不能承诺修复、祛痘、治疗。

### 15. 口腔诊所

```text
我做口腔诊所，想拍种植牙内容拿预约。
```

期望：讲评估边界、流程透明、材料清单，不承诺疗效、价格和成功率。

### 16. 法律咨询

```text
我是婚姻律师，想拍短视频获得咨询。
```

期望：给判断标准和材料准备，不替代法律意见，不承诺结果。

### 17. 财税服务

```text
我做小微企业财税代账，想拍视频获客。
```

期望：用户想判断“我的风险在哪、服务靠不靠谱”，不承诺节税结果。

### 18. 教培招生

```text
我做初中数学辅导，想通过视频招生。
```

期望：高风险表达，不能承诺提分；先讲适合谁、不适合谁和学习过程。

### 19. 老账号有播放无私信

```text
我发了半年，有几条播放还行，但私信很少。
```

期望：不当新号处理，先要 9 条历史内容，判断是围观用户、信任还是行动障碍。

### 20. 编导帮客户

```text
我是短视频编导，客户是做建材的老板，他只想让我给脚本，不想讲用户。
```

期望：识别服务方身份，目标用户仍是客户的客户；输出能让编导转述给客户的解释。
