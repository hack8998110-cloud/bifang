# 毕方发布后数据复盘压测记录

> 日期：2026-07-19  
> 目标：验证 `bifang-feedback` 是否能根据数据反推“下一条该拍什么”，而不是只点评数据好坏。  
> 结论：原有规则能粗略判断卡点，但不够编导化；需要把数据映射回用户观看链路：没停下、没看完、没相信、没行动、吸引错人。

## 压测标准

合格复盘必须输出：

- 数据卡点对应的用户行为判断。
- 原脚本/选题可能丢了哪一个编导锚点。
- 下一条应该拍什么，而不只是“优化开头/增强 CTA”。
- 需要补什么证据。
- 如果数据不足，要标注不确定，不硬判。

## Case 1：完播低

输入：

```text
平台：抖音
行业：系统门窗
选题：买系统门窗别先问价格，先看这 3 个验收细节
播放量：3200
2 秒跳出：42%
5 秒完播：31%
完播率：8%
点赞：18
评论：2
私信：0
账号近期平均：播放 2800，完播 18%
```

合格复盘：

```markdown
## 数据复盘结论

**判定**：需要改开头和中段承诺，不是重做行业方向。

### 最大问题
这条不是完全没人看，而是用户前 5 秒没有确认“跟我家装修有什么关系”，中段也没有持续给验收判断。

### 数据说明
- 曝光：略高于账号均值，说明选题方向不差。
- 2 秒跳出：偏高，第一眼/第一句没有让装修业主快速认领。
- 5 秒完播和整体完播：明显低于均值，说明开头承诺不清，中段可能在讲概念或铺垫。
- 转化：私信为 0，信任证据和低压承接都没建立。

### 下一条怎么改
1. 下一条不要继续泛讲“系统门窗避坑”，改拍“验收现场：这条缝超过多少就要问清楚”。
2. 开头用现场画面，不用口播铺垫：先拍窗框缝隙/密封条/五金，再说判断标准。
3. 结尾不要直接私信报价，改成“把你家窗户问题发来，先判断该重点看密封、五金还是安装”。

### 可沉淀规则
门窗类内容有曝光但完播低时，优先把抽象参数改成现场可判断细节。
```

判定：通过标准。

## Case 2：播放高但无私信

输入：

```text
平台：抖音
行业：烧烤店
选题：烧烤好不好吃，别先看装修，先看这 3 个出餐细节
播放量：58000
完播率：24%
点赞：1100
评论：96
收藏：23
转发：31
私信：0
账号近期平均：播放 6000，完播 19%
```

合格复盘：

```markdown
## 数据复盘结论

**判定**：内容能吸引围观，但没有把围观转成到店意图。

### 最大问题
用户看懂了“烧烤出餐细节”，但没被引导到“我今晚怎么来、怎么点、多少钱”这个行动判断。

### 数据说明
- 曝光：显著高于均值，说明开头和话题有传播。
- 完播：高于均值，说明内容节奏成立。
- 互动：点赞评论高，但收藏低，说明情绪和围观强，实用决策弱。
- 转化：私信为 0，承接动作和到店门槛缺失。

### 下一条怎么改
1. 下一条拍“2 个人第一次来，按 80/120/160 三档怎么点”，把围观转成行动。
2. 画面证据要给真实菜单、桌面菜量、结账金额，别只拍烤串特写。
3. 结尾承接不要问“想吃吗”，改成“评论人数和预算，我给你配一桌不浪费的”。

### 可沉淀规则
餐饮高播放无私信，不一定是内容失败，往往是行动门槛没拆清：价格、人数、距离、点单方式。
```

判定：通过标准。

## Case 3：私信有但成交弱

输入：

```text
平台：视频号
行业：企业增长咨询
选题：增长没起色先别投流，先看这 3 个卡点
播放量：4200
完播率：36%
点赞：86
评论：14
收藏：51
私信：19
成交：0
私信内容：大多问“怎么收费”“适合什么企业”“能不能保证效果”
账号近期平均：播放 3000，私信 5
```

合格复盘：

```markdown
## 数据复盘结论

**判定**：内容能建立初步兴趣，但成交前信任和筛选边界不清。

### 最大问题
用户愿意咨询，说明选题和价值感成立；但私信问题集中在价格、适配和效果保证，说明下一条要拍“适合谁/不适合谁/怎么判断能不能做”，不是继续加大投流或写更硬 CTA。

### 数据说明
- 曝光：高于均值，方向可继续。
- 完播和收藏：较好，方法密度有效。
- 私信：明显高于均值，承接成立。
- 成交：0，销售链路前的信任边界和适配标准不足。

### 下一条怎么改
1. 下一条拍“这 3 类企业先别做增长咨询，先把内部问题补齐”，先筛选客户。
2. 正文要给适配判断：客单价、销售周期、线索承接、老板是否能配合。
3. 结尾不要承诺效果，改成“发你的行业、客单价和当前线索来源，我先判断是不是咨询能解决的问题”。

### 可沉淀规则
私信多但成交弱，下一条先补适配边界和信任证据，不要继续放大承诺。
```

判定：通过标准。

## 发现的问题

原 `bifang-feedback` 缺三块：

1. 没有把数据映射成用户行为：没停、没看完、没信、没行动、吸引错人。
2. 没有要求输出“下一条该拍什么”，容易停留在“优化开头/优化 CTA”。
3. 没有区分“播放高无私信”和“私信多无成交”这两种商业问题。

## 需要写入 feedback 的规则

- 2 秒跳出高：优先反推第一眼画面、第一句、目标用户识别。
- 5 秒完播低：优先反推开头承诺是否清楚。
- 完播低：反推中段是否持续给判断标准。
- 播放高互动高但私信低：反推是否泛流量、缺行动门槛。
- 私信高成交低：反推适配边界、信任证据、销售承接。
- 复盘结论必须给下一条内容方向，而不是只给优化动作。
