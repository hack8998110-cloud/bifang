# 编导思维蒸馏后压测样例

> 目的：验证毕方不是直接给拍剪建议，而是先帮助用户判断客户到底想看什么、为什么先拍、证据从哪里来。

## Case 1：餐饮门店

输入：

```text
我是开烧烤店的，想做抖音引流。
```

合格输出应先判断：

- 用户判断题：附近的人不是想看老板介绍，而是想判断今晚值不值得来试、味道和体验会不会踩雷。
- 先拍方向：不要先拍“本店环境好”，先拍“第一次来怎么点不踩雷”“为什么店里越忙越要拍真实出餐过程”“人均多少钱能吃舒服”。
- 证据：出餐过程、真实价格、桌面菜量、排队/翻台、顾客反馈。
- 禁止跑偏：不要直接给剪辑节奏、转场、探店口播套话。

## Case 2：建材门窗

输入：

```text
我做系统门窗，想获客。
```

合格输出应先判断：

- 用户判断题：装修业主想判断怎么避免买错，哪些细节决定隔音、保温、售后和长期体验。
- 先拍方向：第一条优先给验收/避坑判断标准，而不是品牌介绍。
- 证据：型材截面、五金、安装缝隙、密封条、工地现场、售后边界。
- 禁止跑偏：不能虚构隔音数值、节能结果或案例成交数据。

## Case 3：B2B 设备

输入：

```text
我们卖自动化设备，客户都是工厂老板。
```

合格输出应先判断：

- 用户判断题：工厂老板不是想看设备炫技，而是判断这套设备适不适合自己的工况、交付风险多大、售后靠不靠谱。
- 先拍方向：先拍适配边界、改造前要问的 3 个问题、真实交付流程。
- 证据：工况限制、现场流程、交付清单、售后响应边界、可验证参数来源。
- 禁止跑偏：不能编“效率提升 XX%”，不能替客户假设工况。

## Case 4：知识 IP / 编导服务

输入：

```text
我是短视频编导，想帮客户出内容方向。
```

合格输出应先判断：

- 用户判断题：目标用户不是编导本人，而是客户的客户；编导要能向客户解释为什么不先拍卖点。
- 先拍方向：先帮客户梳理用户为什么会看、为什么会信、为什么会行动。
- 证据：客户过往成交问题、私信高频问题、评论区问题、真实案例和素材。
- 转述口径：这条不是少讲卖点，而是先解决用户当前最不放心的判断题；判断题解决了，卖点才有机会被相信。

## 判定

如果输出只给：

- 拍摄清单
- 剪辑节奏
- 爆款标题
- 泛泛避坑
- “建议你多拍老板日常”

但没有用户判断题、先拍理由、判断标准和证据来源，视为不合格。
