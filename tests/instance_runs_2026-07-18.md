# Instance Runs 2026-07-18

本文件使用 `work/bifang-lite/server_data_export` 和 `obsidian_sync/毕方测试数据` 中的历史真实样例，按 `director_thinking_pressure_tests.md` 的 10 分表做人工试跑。

## Case A：本地餐饮 / 生鲜馄饨 / 宝妈

历史输入：

```text
行业：本地餐饮
产品：生鲜馄饨
目标用户：宝妈群体
目标：到店/预约
素材：每日新鲜食材，现包现做
```

新版入口输出应为：

```text
## 1. 你的用户真正想看的
- 用户现在的决策阶段：准备行动，但缺少“给孩子吃是否放心”的信任证据。
- 用户会停下来，因为：宝妈不是想听你说馄饨好吃，而是想确认馅料、制作时间和调味是否看得见。
- 用户愿意看完，因为：看完能知道这家店是不是值得今天带孩子试一餐或预约一份。
- 用户看完要带走的判断：看食材是否摊开、制作是否现包、调味是否透明、到店/预约是否省事。
- 我为什么这么判断：输入里已经有明确产品、目标用户、到店/预约目标和可拍证据，所以先解决行动前的信任门槛。

## 2. 今天先拍这 3 条
1. 馅料里到底放了什么，摊开给宝妈看
   - 解决哪个决策阶段：不信任 -> 准备行动
   - 需要证据：鲜肉、蔬菜、基础调味、现包画面
2. 早上第一锅馄饨，从备料到包好拍一遍
   - 解决哪个决策阶段：正在比较
   - 需要证据：时间、明档、手工包制
3. 下班顺路 5 分钟，孩子晚饭怎么预约
   - 解决哪个决策阶段：准备行动
   - 需要证据：预约流程、取餐路线、到家加热

## 3. 第一条别改丢
- 用户问题：这份馄饨给孩子吃，食材和调味能不能放心？
- 判断标准：食材摊开、调味透明、现场包制、到店可见。
- 信任证据：每日食材、明档/后厨、现包现做。
- 成交理由：宝妈看懂后更愿意先试一份或预约。
- 承接动作：评论/私信问“孩子几岁、口味偏淡还是正常”，再给试吃或预约建议。
```

评分：9/10

- 扣分：缺少具体门店距离、客单价和预约方式，只能基于默认假设判断行动门槛。

## Case B：三农 / 平谷大桃 / 自家果园

历史输入：

```text
行业：三农行业
产品：平谷大桃
目标用户：喜欢吃水果的人
目标：成交转化
素材：自己家的果园，随时可以在果园拍
信任：自家果园，只卖自己的，卖完就没有
顾虑：桃子不够甜、不够好吃、货不对板
```

新版入口输出应为：

```text
## 1. 你的用户真正想看的
- 用户现在的决策阶段：不信任，但客单不高，信任一过就能准备行动。
- 用户会停下来，因为：网购水果最怕视频里好看，到手不甜、不新鲜、货不对板。
- 用户愿意看完，因为：看完能知道这箱桃是不是从真实果园摘、怎么判断成熟度、售后边界是什么。
- 用户看完要带走的判断：看产地、采摘标准、成熟度证据、装箱规格和坏果处理方式。
- 我为什么这么判断：素材里“自家果园”和“随时可拍”是最强信任证据，顾虑集中在甜度和货不对板。

## 2. 今天先拍这 3 条
1. 今天这批桃从哪棵树摘，装箱前给你看
   - 解决哪个决策阶段：不信任
   - 需要证据：果园实拍、采摘、分拣、装箱
2. 树上熟和没熟透的桃，切开差在哪
   - 解决哪个决策阶段：正在比较
   - 需要证据：切面、汁水、香味、当天真实成熟度
3. 60-80 元一箱到底装什么，坏果怎么处理
   - 解决哪个决策阶段：准备行动
   - 需要证据：规格、重量、包装、售后边界

## 3. 第一条别改丢
- 用户问题：我会不会买到和视频不一样的桃？
- 判断标准：产地可见、采摘可见、分拣可见、规格可见、售后边界清楚。
- 信任证据：自家果园、当天采摘装箱、只卖自家库存。
- 成交理由：货不对板风险降低后，用户才愿意私信问规格和链接。
- 承接动作：评论/私信问“要几斤装、几天能收、能不能接受自然口感波动”。
```

评分：8/10

- 扣分：目标用户“喜欢吃水果的人”仍偏宽，应该进一步分成家庭买家、送礼买家、复购买家。
- 历史旧输出问题：风险词曾混入“根治/治疗”等非水果行业套话；新版必须打回这种行业污染。

## Case C：洗护 / 冷制皂 / 非遗传承人

历史输入：

```text
行业：洗护行业
产品：冷制皂
目标用户：想使用天然无添加洗护用品的人
目标：成交转化
素材：过敏性皮肤群体、根据皮肤定制洗护用品、中药浸泡液
信任：非遗古法传承人
差异化：半年以上中药浸泡油
限制：规避洗护和美业违规词
```

新版入口输出应为：

```text
## 1. 你的用户真正想看的
- 用户现在的决策阶段：正在比较，同时不信任“天然无添加”这类泛卖点。
- 用户会停下来，因为：用户不是想听冷制皂多天然，而是想知道怎样判断一块皂是不是值得试、会不会过度刺激。
- 用户愿意看完，因为：看完能拿到工艺、成分来源和适用边界的判断标准。
- 用户看完要带走的判断：看油脂来源、熟成/浸泡时间、制作过程、使用场景和不能承诺什么。
- 我为什么这么判断：输入有强差异化和信任背书，但洗护合规风险高，不能先做功效承诺。

## 2. 今天先拍这 3 条
1. 半年浸泡油开坛，不讲功效，只讲工艺差在哪
   - 解决哪个决策阶段：不信任
   - 需要证据：陶罐日期、油色、开坛、入皂过程
2. 冷制皂怎么判断是不是皂基重塑
   - 解决哪个决策阶段：正在比较
   - 需要证据：配方记录、搅拌、入模、熟成
3. 第一次试冷制皂，哪些情况先别乱换
   - 解决哪个决策阶段：准备行动前的风险边界
   - 需要证据：使用边界、清洁场景、个体差异提示

## 3. 第一条别改丢
- 用户问题：这块皂是不是又一个“天然”概念？
- 判断标准：浸泡时间可见、工艺可见、边界清楚、不承诺治疗修复。
- 信任证据：非遗身份、陶罐日期、制作过程。
- 成交理由：用户看见时间成本和边界后，更愿意私信问适合哪类清洁场景。
- 承接动作：评论/私信问“日常洗脸/洗澡/洗手哪个场景”，不问医学肤质诊断。
```

评分：8/10

- 扣分：历史输入里的“过敏性皮肤群体/根据皮肤定制”风险很高，必须改成感受式描述；不能直接承接“肤质诊断”。
- 历史旧输出问题：曾出现“敏感肌/过敏肌/帮你配一套”等承诺感较强表达，审核需要更严。

## Case D：手工皂 / 冷制皂和洁面皂 / 私信咨询

历史输入：

```text
行业：手工皂
产品：手工冷制皂和洁面皂
目标用户：想温和清洁但害怕踩坑的女性用户
目标：私信咨询
素材：真实制作过程、配方记录、用户反馈、工作室场景
```

新版入口输出应为：

```text
## 1. 你的用户真正想看的
- 用户现在的决策阶段：正在比较，不确定手工皂是真是假、适不适合日常清洁。
- 用户会停下来，因为：用户想知道怎么分辨冷制慢工和皂基重塑，而不是听品牌说自己温和。
- 用户愿意看完，因为：看完能学会看工艺、配方记录、熟成时间和使用边界。
- 用户看完要带走的判断：真假冷制看过程，适不适合看使用场景，不靠功效承诺判断。
- 我为什么这么判断：输入给了制作过程和配方记录，这是最适合先建立判断标准的素材。

## 2. 今天先拍这 3 条
1. 同样叫手工皂，冷制和皂基重塑怎么一眼看
   - 解决哪个决策阶段：正在比较
   - 需要证据：称油、搅拌、入模、晾皂
2. 洗完紧绷，不一定是皂不行，可能是场景没选对
   - 解决哪个决策阶段：没意识到问题/正在比较
   - 需要证据：不同皂型、洗感说明、使用边界
3. 客户退回一批皂后，我改了什么
   - 解决哪个决策阶段：不信任
   - 需要证据：脱敏反馈、旧配方/新配方、改配方过程

## 3. 第一条别改丢
- 用户问题：我怎么判断这不是皂基重塑的概念货？
- 判断标准：称料、搅拌、入模、脱模、熟成记录。
- 信任证据：真实制作过程、配方记录、工作室场景。
- 成交理由：用户学会判断后，更愿意私信问具体清洁场景。
- 承接动作：私信问“你主要用于洗脸、洗手还是沐浴”，不做皮肤问题诊断。
```

评分：9/10

- 扣分：还需要真实 SKU、价格和私信承接话术，才能判断第三条是否过早进入信任深水区。

## 共性失败点

旧网页版输出暴露出这些问题，必须写入 skill 规则：

1. `watchReason` 为空却仍判通过。
2. 默认输出 `viralElement/score`，容易把客户拉回“爆款元素”和评分表。
3. 行业风险词库污染：水果样例混入护肤/医疗式风险词。
4. 部分 CTA 过硬，如“扣1/私信我发链接”，没有先做筛选问题。
5. 洗护/手工皂容易越界到肤质诊断、功效承诺或特殊人群暗示。

## 本轮结论

当前新版规则在这 4 个历史样例上，人工试跑均达到 8 分以上。  
但为了让模型执行更稳定，还需要把“旧输出兼容审核”写进 `topic/review/report`：遇到空观看理由、旧爆款字段、行业风险污染时必须打回。
