# 毕方正向行业入口回归用例

> 用途：验证毕方不只是会拦错，还能在低信息输入下稳定启发客户“用户到底想看什么、先拍什么、为什么先拍”。

## 验收标准

每个样例必须输出：

- 用户现在的决策阶段。
- `行业翻译成的用户判断题`，且不是泛泛“避坑/干货/专业内容”。
- 今天先拍 3 条，且至少覆盖“比较/信任/行动”中的两类。
- 第一条别改丢的用户问题、判断标准、信任证据、成交理由、承接动作。
- 不输出完整脚本、拍剪清单、评分表、爆款元素。

## 8 个正向样例

| ID | 低信息输入 | 用户真正想看的判断题 | 第一条优先方向 |
|---|---|---|---|
| PS-01 | 我开社区面馆，想拍抖音引流到店 | 今天值不值得专门来吃一碗，味道、分量、距离会不会踩雷 | 先拍一碗面从出锅到端上桌，给价格、分量、口味选择 |
| PS-02 | 我卖系统门窗，想做小红书 | 装修时怎么判断门窗别买错，哪些细节影响隔音、保温和售后 | 先拍业主最容易忽略的 3 个验收细节 |
| PS-03 | 我做工业除尘设备，想获客 | 设备能不能适配我的工况，供应商是不是只会套方案 | 先拍一个工况如何判断风量、粉尘、管道和安装边界 |
| PS-04 | 我做企业增长咨询，想拿线索 | 这个咨询师能不能先判断我的增长卡点，而不是只讲理论 | 先拍“增长没起色先别投流，先看这 3 个卡点” |
| PS-05 | 我做手工银饰定制 | 这件东西为什么值这个价，审美和交付会不会翻车 | 先拍一件定制从需求到草图到成品的判断过程 |
| PS-06 | 我开宠物店，想做附近洗护复购 | 宠物在这家店会不会被粗暴对待，卫生和细节靠不靠谱 | 先拍一次洗护前检查、安抚、工具消毒和吹干细节 |
| PS-07 | 我是职场沟通课老师 | 这个方法能不能解决我明天就要遇到的汇报场景 | 先拍一个具体汇报场景的改法，不承诺升职加薪 |
| PS-08 | 我是编导，客户做全屋定制，只想要脚本 | 装修业主怎么判断这家定制不是只会讲板材和价格 | 先拍“柜子好不好，不先看板材，先看这 3 个落地细节” |

## 失败信号

- 把“行业”当成内容方向，没有翻译成用户判断题。
- 3 条都叫避坑、内幕、真相，但解决的是同一个阶段。
- 第一条只有标题，没有说明为什么先拍。
- 给商家自我介绍、团队背书、拍剪建议，而不是给用户判断标准。
- 编导场景下，忘记目标用户是客户的客户。
