# 毕方 V0.2-rc 版本说明

状态：正式候选版。  
日期：2026-07-19  

## 一句话定位

毕方 V0.2-rc 是一套全行业商业短视频编导判断 skills：先判断用户为什么看、该先拍什么、为什么相信、怎么行动，再进入选题、脚本、审核、改稿、复盘和报告。

它不是拍剪建议工具，也不是标题库。它的核心价值是帮使用者建立编导思维：

```text
行业/产品 -> 用户正在判断的问题 -> 先拍顺序 -> 信任证据 -> 承接动作 -> 发布后复盘
```

## V0.2-rc 解决了什么

1. 低信息输入不直接写脚本，先输出用户判断题、先拍 3 条和改稿锚点。
2. 真实行业不靠模板硬编，先把行业翻译成用户决策问题。
3. 混合账号能分清消费决策、人格信任和交付证据。
4. 老账号能用 9 条历史内容判断高播放、高私信和无效常发之间的偏差。
5. 客户改稿时能识别用户问题、判断标准、信任证据、成交理由、承接动作被改丢。
6. 高风险完整脚本能一票否决承诺结果、恐吓、诊断、虚构案例和高压承接。
7. 交付报告必须标注交付状态，不能把缺证据内容包装成可交付成果。

## 主要模块

- `bifang-starter`：极简入口，默认回答用户真正想看什么、今天先拍什么、第一条别改丢什么。
- `bifang-diagnosis`：商业诊断，判断观看理由、成交理由、信任来源和转化路径。
- `bifang-profile`：账号画像，沉淀后续选题和脚本可复用字段。
- `bifang-topic`：选题生成，每条选题必须说明用户判断题、观看理由、成交理由、可信证据。
- `bifang-script`：脚本撰写，保留判断标准、信任证据、承接动作和改稿锚点。
- `bifang-review`：脚本审核，严审用户观看闸门、商业闸门、改稿闸门和高风险表达。
- `bifang-rewrite`：改稿，增强表达但不改丢 5 个锚点。
- `bifang-feedback`：发布后复盘，支持单条数据和老账号 9 条内容复盘。
- `bifang-report`：交付报告组装，必须标注交付状态和证据缺口。

## 开源版边界

开源版包含通用方法、通用 skills、通用测试样例和干净文档。

开源版不包含：

- 私有达人/竞品研究资料。
- 客户真实数据和未脱敏案例。
- 内测销售资料、报价资料、私有行业资产。
- 私有账号压测明细和内部交付记录。

## 仍需人工注意

- 毕方不会替代真实专业判断。法律、财税、医疗、教培、留学移民等高风险行业必须保留边界。
- 缺少真实案例、现场、数据、资质时，只能标注待补证据，不能写成事实。
- 客户改稿后必须重新审核，不能因为原稿通过就默认改稿可发。
- 发布后必须用真实数据复盘，不能只凭脚本质量判断成败。

## 是否可以进入 V0.2 正式版

可以进入 V0.2 正式版前的最后整理阶段。  
正式发布前只建议做三件事：

1. 确认开源包无内测资料。
2. 确认 README、发布说明、验收清单口径一致。
3. 用 1-2 个真实用户再跑一次安装和首次使用流程。
